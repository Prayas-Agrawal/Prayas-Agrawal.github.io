Microkernel emulation for Linux

(probably the coolest systems project I have done) I was interested in checking if linux could have support for microkernel like device drivers like in Zircon kernel (or L4). I developed a kernel module for providing such support. It worked by providing the user space driver direct DMA access (well not direct, but mostly) for fast access to hardware. I preloaded the PCI memory pages with surrogate pages pointing to memory managed by my module. But the simplest and most exciting part was how I came up with a Kernel writethrough but read opaque mechanism which eliminated memory copies when when transitioning from userland to kerneland (and vice versa).

I also made a syscall arbiter which dynamically rerroutes the syscall to the userland driver or the kernel implementation based on users prefs

I tested it out on an RTL8139, made a simple ICMP query. Tests passed.
